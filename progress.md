## Architecture Review Notes (2026-03-11)

- 현재 구조는 `core / features / domain / platform / ui / legacy`로 분리되어 있으나, 실행 모델 기준으로는 `GS` 싱글턴, `GAME` legacy bridge, `deps_factory`, `modules/fns` registry가 동시에 남아 있어 진입 경로가 중복된다.
- 대표 병목:
  - `game/core/game_state.js`: 전역 state + dispatch + event bus + business method facade가 한 객체에 합쳐져 있음.
  - `game/platform/legacy/global_bridge_runtime.js`: state/data/audio/modules/api를 한 번 더 전역 브리지로 노출함.
  - `game/features/combat/app/combat_actions.js`: feature app layer가 실제로는 UI facade/router 역할까지 넓게 담당함.
  - `game/ui/screens/event_ui_flow.js`, `game/ui/run/run_setup_helpers.js`, `game/ui/title/character_select_ui.js`: UI가 system/data/core 상태 초기화와 비즈니스 규칙을 직접 안다.
  - `game/systems/run_rules.js`, `game/domain/combat/turn/turn_state_mutators.js`, `game/combat/death_handler.js`: 직접 상태 변경이 집중된 hot spot.
- 저장된 메트릭 기준 현재 주요 횡단 의존성은 `core->ui 43`, `ui->data 38`, `ui->systems 12`, 직접 상태 변경 총량은 `234`다.
- 실무 우선순위:
